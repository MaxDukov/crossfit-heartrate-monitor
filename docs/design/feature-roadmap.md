# Дорожная карта функций (Feature Roadmap)

> Основано на анализе конкурентов (Myzone, Orangetheory, WHOOP, SugarWOD, BTWB, Polar).
> Дата: 24.07.2026.

---

## Приоритет 1 — MVP+ (быстрый выигрыш, 1-3 дня каждый)

### 1.1. Время в зонах (Zone Time)

| Параметр | Значение |
|---|---|
| **Сложность** | Низкая |
| **Время** | 1-2 дня |
| **Вдохновение** | Orangetheory Splat Points, Myzone MEPs |
| **Зависимости** | Нет |

**Что делать:**
- Backend: накапливать секунды в каждой зоне в `_on_hr_data` callback (in-memory dict, сброс при новой сессии)
- WebSocket: добавить `zone_time` в payload
- Frontend: горизонтальный стек-бар на карточке (4 сегмента цветов зон)
- Frontend: время М:СС под каждым сегментом
- Сброс при начале новой сессии

---

### 1.2. Цветной фон карточки по зоне — ✅ Выполнено

| Параметр | Значение |
|---|---|
| **Сложность** | Низкая |
| **Время** | 0.5 дня |
| **Вдохновение** | Myzone Live Tiles, Orangetheory |
| **Зависимости** | Нет |
| **Статус** | **Выполнено** (feature/zone-colored-cards, 24.07.2026) |

**Что сделано:**
- AthleteCard: градиентный фон с rgba-цветом зоны (18-22% light / 30% dark)
- CSS transition 0.8s для плавной смены цвета
- Цвета зон: `#0a8a06` (зелёный), `#f5e505` (жёлтый), `#DC2626` (красный)
- Реализация в `frontend/src/lib/zones.ts` и `frontend/src/components/AthleteCard.tsx`

---

### 1.3. Расчёт калорий

| Параметр | Значение |
|---|---|
| **Сложность** | Низкая |
| **Время** | 0.5 дня |
| **Вдохновение** | Orangetheory, Polar, Myzone |
| **Зависимости** | Нет |

**Что делать:**
- Формула Keyt: `ккал/мин = (0.6309 x HR + 0.1988 x вес + 0.2017 x возраст - 55.0963) / 4.184`
- Упрощённая (без веса/возраста): `ккал/мин = HR x 0.014` (приближение)
- Добавить поля weight и age в модель Athlete (опционально)
- WebSocket: `calories` в payload
- Frontend: показать ккал на карточке

## Приоритет 2 — Вовлечённость (1-2 недели)

### 2.1. Баллы усилий (Effort Points)

| Параметр | Значение |
|---|---|
| **Сложность** | Средняя |
| **Время** | 3-5 дней |
| **Вдохновение** | Myzone MEPs (1-4 балла/мин по зонам) |
| **Зависимости** | 1.1 Zone Time |

**Что делать:**
- Начислять баллы по зонам: Z1=1, Z2=2, Z3=3, Z4=4 за минуту
- Сохранять в БД (новая таблица `EffortPoints` или колонки в `SessionAthlete`)
- Накопительный баланс per-athlete (за всё время, за месяц)
- Frontend: показать баллы на карточке + в аналитике
- Отдельная страница "Рейтинг" (лидерборд по баллам за месяц)

---

### 2.2. Достижения и бейджи

| Параметр | Значение |
|---|---|
| **Сложность** | Средняя |
| **Время** | 3-5 дней |
| **Вдохновение** | BTWB PR-празднования, Myzone статусы (Iron-Gold) |
| **Зависимости** | 2.1 Effort Points |

**Что делать:**
- Статусы: Новичок - Бронза - Серебро - Золото (по сумме баллов)
- Бейджи: "100 баллов за сессию", "7 дней подряд", "Рекорд зоны", "Марафон" (300 баллов/мес)
- Frontend: анимация при получении нового бейджа
- Отдельная страница "Профиль" с бейджами

---

### 2.3. Сравнение с прошлой сессией

| Параметр | Значение |
|---|---|
| **Сложность** | Низкая |
| **Время** | 1-2 дня |
| **Вдохновение** | WHOOP, Polar |
| **Зависимости** | Нет |

**Что делать:**
- Backend: при завершении сессии сравнить avg HR, zone distribution, calories с предыдущей
- Показать дельту на странице аналитики
- Цветовая индикация: зелёный (улучшение), красный (ухудшение)

## Приоритет 3 — Рост (2-4 недели)

### 3.1. Экспорт во внешние сервисы

| Параметр | Значение |
|---|---|
| **Сложность** | Средняя |
| **Время** | 5-7 дней |
| **Вдохновение** | Myzone (Strava, Apple Health, Garmin) |
| **Зависимости** | Нет |

**Что делать:**
- **Strava**: OAuth2, POST activity (TCX/FIT файл с HR данными)
- **Apple Health**: HealthKit REST (для iOS-приложения)
- **Google Fit**: REST API (для Android)
- Настройка в профиле спортсмена: "Привязать Strava"
- Авто-экспорт при завершении сессии

---

### 3.2. Telegram-бот

| Параметр | Значение |
|---|---|
| **Сложность** | Низкая |
| **Время** | 2-3 дня |
| **Вдохновение** | Российский рынок |
| **Зависимости** | Нет |

**Что делать:**
- Бот на python-telegram-bot
- Команды: /start (привязка аккаунта), /stats (статистика за неделю), /session (результат последней сессии)
- Push при завершении сессии: "Сессия завершена. Средний пульс 145, 23 мин в зелёной зоне, 320 ккал"
- Групповой чат зала: ежедневный лидерборд по баллам усилий

---

### 3.3. Аналитика для тренера/владельца

| Параметр | Значение |
|---|---|
| **Сложность** | Высокая |
| **Время** | 1-2 недели |
| **Вдохновение** | Myzone Admin Portal (at-risk detection, +33% визитов) |
| **Зависимости** | 1.1 Zone Time, 1.3 Калории |

**Что делать:**
- Дашборд "Обзор зала": посещаемость, средний пульс по группам, популярные временные слоты
- At-risk detection: спортсмены с падением частоты тренировок >30%
- Тренды: сравнение месяцев по вовлечённости, удержанию, баллам
- Экспорт отчётов в CSV/PDF

---

### 3.4. Поддержка BLE-датчиков (BYOD)

| Параметр | Значение |
|---|---|
| **Сложность** | Высокая |
| **Время** | 2-3 недели |
| **Вдохновение** | Myzone Go, Polar 360 |
| **Зависимости** | Нет |

**Что делать:**
- Мобильное приложение (React Native / Flutter) или PWA
- Приложение принимает BLE HR с часов/пояса спортсмена
- Передаёт на backend через WebSocket (телефон как ретранслятор)
- Альтернатива: Web Bluetooth API для Chrome на том же Pi

```
Фаза 1 (1 неделя):   1.2 -> 1.1 -> 1.3
Фаза 2 (3 недели):   2.1 -> 2.3 -> 2.2
```
