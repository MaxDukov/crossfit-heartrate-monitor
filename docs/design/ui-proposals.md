> Основано на анализе Myzone, Orangetheory, BTWB, Polar Club.
> Дата: 24.07.2026.

## Предложение 1. Цветной фон карточки по зоне

### Проблема
Сейчас цвет зоны отображается только на цифрах пульса и слабом градиенте фона. С расстояния 3-5 м (типичное для зала) сложно мгновенно определить зону спортсмена.

### Решение
Вся карточка заливается цветом активной зоны с прозрачностью ~15-25%. Цифры пульса остаются крупными и контрастными.

### Вдохновение
- **Myzone Live Tiles** — вся плитка окрашена в цвет зоны (grey/blue/green/yellow/red)
- **Orangetheory** — вся строка меняет цвет фона при переходе между зонами

### Макет

```
 Зона 3 (Amber)               Зона 2 (Green)
┌───────────────────────────┐ ┌───────────────────────────┐
│  ██████████████████████   │ │  ██████████████████████   │
│  ██     Максим     ████   │ │  ██     Никита    ████   │
│  ██                  ████ │ │  ██                  ████ │
│  ██      165        ████  │ │  ██      142        ████ │
│  ██                  ████ │ │  ██                  ████ │
│  ██  Высокая  87%  ████   │ │  ██  Умеренная  74% ████ │
│  ██████████████████████   │ │  ██████████████████████   │
│  [график истории пульса]  │ │  [график истории пульса]  │
└───────────────────────────┘ └───────────────────────────┘
```

### Технические детали
- `background-color` карточки = цвет зоны с `opacity: 0.15` (light) / `0.25` (dark)
- Сохранить текущий градиентный оверлей поверх для глубины
- Цвет цифр остаётся контрастным (текущий zoneColor)
- Анимация перехода: `transition: background-color 0.8s ease`

### Ожидаемый эффект
Мгновенная читаемость состояния группы с расстояния — тренер видит «карту зала» за 1 секунду.

## Предложение 2. Время в зонах (Zone Time bar)

### Проблема
Текущий `% от максимума` — абстрактная метрика. Спортсменам и тренеру нужен понятный ориентир: сколько времени проведено в каждой зоне за сессию.

### Решение
Компактный горизонтальный бар внизу карточки, показывающий распределение времени по зонам за текущую сессию.

### Вдохновение
- **Orangetheory Splat Points** — счётчик минут в оранжевой/красной зоне
- **Myzone MEPs** — накопительные баллы за время в зонах
- **Strava Activity Zones** — распределение времени по HR-зонам

### Макет

```
┌───────────────────────────┐
│        Никита             │
│                           │
│         142              │
│                           │
│  Умеренная  74%  Max 195  │
│                           │
│  З1    З2     З3    З4    │
│  ████░ ████████ ███░ ░░░░ │
│  3:20   8:15   2:40  0:00 │
│                           │
│  [график истории пульса]  │
└───────────────────────────┘
```

### Технические детали
- Накапливать секунды в каждой зоне (backend: в `_on_hr_data` callback)
- Отправлять в WebSocket payload: `"zone_time": {"1": 200, "2": 495, "3": 160, "4": 0}`
- Фронтенд: горизонтальный стек-бар (4 сегмента, цвет = цвет зоны)
- Под баром — время в формате М:СС
- Сброс при начале новой сессии

### Ожидаемый эффект
- Тренер видит, кто «недорабатывает» (много времени в зоне 1) и кто «перегружает» (много в зоне 4)
- Спортсмены получают визуальную цель — «провести 10 минут в зелёной зоне»

## Предложение 3. Плавные переходы при изменении состава

### Проблема
При подключении/отключении датчика карточки мгновенно появляются/исчезают, остальные перескакивают на новые позиции. На TV это выглядит дёргано.

### Решение
CSS-анимации для появления/исчезновения карточек + плавная перестройка grid.

### Вдохновение
- **BTWB Workout TV** — плавные переходы между режимами экрана
- **macOS Stage Manager** — плавное перераспределение окон

### Технические детали
- Новая карточка: `animation: fadeInScale 0.5s ease` (появление с масштабом 0.9 -> 1.0)
- Уходящая карточка: `animation: fadeOut 0.5s ease` (исчезновение, затем grid перестраивается)
- Grid: `transition: grid-template-columns 0.4s ease`
- Задержка удаления: держать «призрак» 2 сек перед фактическим удалением из DOM
- Для React: использовать `AnimatePresence` из framer-motion или CSS-only через `onAnimationEnd`

### Ожидаемый эффект
- Профессиональный вид экрана при динамическом составе
- Нет «скачков» при временном пропадании сигнала датчика
